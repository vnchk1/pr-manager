## Основные настройки запуска

```
run:
  go: '1.24'           # Версия Go для анализа
  concurrency: 4       # Количество параллельных горутин
  issues-exit-code: 1  # Возвращать код 1 при найденных проблемах
  tests: true          # Анализировать тесты
  modules-download-mode: mod
  allow-parallel-runners: true
  ```

## Вывод результатов

```
output:
  formats:
    - format: colored-line-number  # Цветной вывод с номерами строк
  print-issued-lines: false        # Не печатать проблемные строки
  print-linter-name: true          # Показывать имя линтера
  sort-results: true              # Сортировать результаты
  sort-order:
    - linter
    - severity
    - file
```
## Настройки линтеров

### Включение/отключение линтеров

```
linters:
  enable-all: true      # Включить все линтеры
  disable:              # Исключить конкретные линтеры
    - exhaustruct      # Проверка заполнения структур
    - nonamedreturns   # Именованные возвращаемые значения
    - wrapcheck        # Обертки ошибок
    - gofumpt          # Форматирование (альтернатива gofmt)
    - gci              # Сортировка импортов
    # ... и другие
```
## Специфические настройки линтеров

### gosec (безопасность)

```
gosec:
  excludes:
    - G401  # Использование weak crypto
    - G505  # SHA1 хэширование
    - G306  # Права файлов
    - G115  # Использование context.WithValue
```
### stylecheck

Включены только определенные проверки стиля кода из серии ST1000+

### Проверка орфографии

```
misspell:
  ignore-words:
    - sub_payed  # Игнорировать это слово
```
### Длина строк

```
lll:
  line-length: 140  # Максимальная длина строки
```
### Контроль зависимостей

```
depguard:
  rules:
    main:
      list-mode: lax
      allow:
        - $gostd        # Стандартная библиотека
        - filippo.io/edwards25519  # Разрешенные сторонние пакеты
        - golang.org/x/crypto
      deny:
        - pkg: io/ioutil  # Запрещенный устаревший пакет
```

## Исключения

```
issues:
  exclude-rules:
    - linters:
        - lll
      source: ".*// "  # Игнорировать комментарии для lll
  max-issues-per-linter: 0  # Без ограничений
  max-same-issues: 0
```
## Назначение конфигурации

Данная конфигурация создает строгий процесс линтинга с:

- Проверкой безопасности (gosec)
- Анализом сложности кода
- Проверкой стиля
- Контролем зависимостей
- Проверкой орфографии

При этом исключены некоторые линтеры, которые могут быть слишком строгими или конфликтовать с другими инструментами разработки.
